## Requirements Gathering Process

```mermaid
sequenceDiagram
    participant S as Stakeholder
    participant BA as Business Analyst
    participant PM as Project Manager
    participant Dev as Development Team
    
    S->>BA: Provide Requirements
    BA->>BA: Analyze Requirements
    BA->>PM: Draft Requirements Doc
    PM->>Dev: Technical Review
    Dev-->>PM: Feasibility Feedback
    PM-->>BA: Revisions Needed
    BA-->>S: Clarification Questions
    S-->>BA: Clarifications
    BA->>PM: Updated Requirements
    PM->>S: Sign-off Request
```
